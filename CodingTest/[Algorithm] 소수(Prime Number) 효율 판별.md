# [Algorithm] 소수(Prime Number) 효율 판별

# 소수(Prime Number) 효율적으로 판별하기

## N에 대해 소수인지 판별할때, N의 제곱근까지만 확인하기

### 모든 약수는 짝(Pair)이 있다.

어떤 수 $N$이 약수를 가진다면, 그 약수들은 반드시 대칭을 이룬다.

예를 들어 $N = 16$일 때 약수는 $(1, 16), (2, 8), (4, 4)$이다.

- 1 * 16  = 16
- 2 * 8 = 16
- 4 * 4 = 16
- 8 * 2 = 16
- 16 * 1 = 16

따라서 $\sqrt{N}$까지만 나누어떨어지는지 확인하면 그 이후는 검사할 필요가 없다.

### 왜 제곱근까지 보면 되는가?

어떤 수 $N$이 약수를 가진다면, $a \times b = N$로 표현이 가능하다.

만약 $a > \sqrt{N}$ 이고 $b > \sqrt{N}$ 이라면, 두 수의 곱인 $a \times b$ > $N$ 이다.

허나 가정이 $a \times b = N$ 이기 때문에 이는 모순이다.

따라서 최소 한개의 수는 무조건 $\sqrt{N}$ 이하여야 한다.

즉, 약수가 존재한다면 **반드시** $\sqrt{N}$ **이하의 영역에 최소한 하나는 존재한다.**

### 코드 예시

```java
public static boolean isPrime(long n) {
		// 1 이하의 수는 소수가 아님
    if (n <= 1) return false;

    // 2부터 시작해서 1씩 증가
    for (long i = 2; i * i <= n; i++) {
        if (n % i == 0) {
            return false;
        }
    }
    return true;
}
```
